# 后续扩展清单

> 本文档记录项目后续需要实现的功能模块，按优先级排序。
> 每次开发前，先在此文档中确认本次要做的功能，再进入开发。

---

## 🔴 高优先级（近期必做）

### 1. 用户体系

| 功能 | 说明 | 关联页面 |
|------|------|----------|
| 用户登录 | 手机号 + 验证码登录 | 登录页 `/login` |
| 用户注册 | 手机号注册流程 | 注册页 `/register` |
| 个人中心 | 用户信息、我的订单、我的发布 | 个人中心页 `/profile` |
| Token 鉴权 | JWT Token 存储与自动刷新 | 全局 request 拦截器 |

### 2. 首页完善

| 功能 | 说明 |
|------|------|
| 担保交易统计 | 累计交易单数从接口获取 |
| 搜索历史 | 记录用户最近搜索 |
| 轮播广告 | 顶部 Banner 广告位 |

### 3. 商品交易流程

| 功能 | 说明 | 关联页面 |
|------|------|----------|
| 购买下单 | 确认商品 → 下单 → 支付 | 下单页 `/order/confirm` |
| 订单管理 | 买家订单列表、状态跟踪 | 订单页 `/orders` |
| 发布商品 | 卖家填写商品信息并发布 | 发布页 `/publish` |

## 技术债务

| 项 | 说明 | 计划处理时间 |
|----|------|-------------|
| 图片资源 | 当前使用 placeholder，需替换为真实 CDN | 接入 OSS 后 |
| 接口联调 | Mock 数据切换为真实接口 | 后端就绪后 |
| 性能优化 | 图片懒加载、虚拟列表、分包加载 | 上线前 |
| 单元测试 | 补充核心组件和工具函数的测试 | 稳定后 |
| TypeScript 严格模式 | 开启 strict 选项 | 重构期 |

## 扩展检查清单

新增功能开发前，请确认：

- [ ] 功能需求是否明确？
- [ ] 是否有设计稿或参考截图？
- [ ] 后端接口是否已定义？
- [ ] 是否需要新增路由？
- [ ] 是否需要新增 Store？
- [ ] 是否需要新增公共组件？
- [ ] 是否影响现有页面？
- [ ] 是否需要更新本文档？
